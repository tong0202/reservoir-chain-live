# 水库链 · 免费河上的真计算（手动点火版）

把两样东西拧成一台：
- **会算的料子（水库）**：随机连好就固定、从不训练的动态网络，只在边上贴一层线性读出。
- **不忘的链（哈希链）**：父哈希环环相扣，可复验、防篡改、断了能接。

每次点火（`workflow_dispatch`），GitHub 的免费服务器跑一个 tick：
读上一状态 → 校验哈希 → 恢复水库 → 现掷一个新随机位 → 水库现场算它的
延迟 XOR（`u[t-2] ^ u[t-5]`）→ 把预测和哈希链写回 `chain_compute/state.json`。

## 这不是记账（关键）

- **答案（正确 XOR）从不写进账本**，账本只存"水库猜了啥"（`prediction_bit`）。
- **每步输入位用"父哈希+步数"当场派生**，账本里不可能预存一个还没掷出的位的答案。
- 本地已用五组对照证伪"记账/查表"：关掉水库 → 掉到 50%；喂从没见过的
  新随机序列 → 仍 100%。记账遇新数据必崩，会算才能对付没见过的东西。

## 边界（不得宣称）

- 真正在跑的是 GitHub 的服务器（一个端点）。它证明"免费的河能驱动链上真计算
  一步、可复验、断了能接"，**不**证明脱离现有硬件、不证明自主创作、不证明生命。
- 水库跑在 numpy 上，是二进制机器在模拟这团动态。
- 本轮纯手动点火，不涉及定时器准不准。

## 看它
- 链状态：`chain_compute/state.json`（每次点火后增长/更新）
- 是不是免费的河跑的：`gh run list --repo <owner>/reservoir-chain-live --json event,conclusion`
